\section{Observed Issues in Existing Compiler/Test Output} %

\begin{marker}\begin{justify}
TODO: Build a short taxonomy with examples.

Suggested issue categories (each with at least one example + explanation of novice difficulty):
\begin{itemize}
    \item \textbf{Signal-to-noise:} essential mismatch buried among stack traces/framework internals.
    \item \textbf{Missing tested expression:} output shows values but not which function call failed (or only indirectly via test name).
    \item \textbf{Inconsistent structure:} different frameworks print the ``same'' concept (failure) in incompatible formats.
    \item \textbf{Non-local information:} to understand failure, one must reconstruct prior operations (data structure sequences).
    \item \textbf{Poor actionability:} message does not suggest what to check next (inputs, edge cases, invariants).
    \item \textbf{Property-based specifics:} counterexample/shrinking exists but is not rendered in a novice-friendly way.
\end{itemize}

Include the didactic constraint explicitly:
\begin{itemize}
    \item Tested expression must be shown without revealing a reference implementation or giving away the solution.
\end{itemize}
\end{justify}\end{marker}

\section{Derived Requirements and Design Implications} %++++
%===========================================================

\begin{marker}\begin{justify}
TODO: Derive requirements as ``R1, R2, ...'' and map each to a design implication.

Candidate requirements (adjust to your final argument):
\begin{itemize}
    \item \textbf{R1 Consistency:} same high-level structure for failures across unit and property-based tests.
    \item \textbf{R2 Minimal actionable core:} always show \emph{tested expression}, \emph{expected}, \emph{actual} as the first screenful.
    \item \textbf{R3 Progressive disclosure:} optional sections for reductions/history/stack traces; novices start with compact view.
    \item \textbf{R4 Expression capture:} render a readable representation of what was tested (with simplification).
    \item \textbf{R5 Value extraction + rendering:} compute and display values in a stable, comparable representation.
    \item \textbf{R6 Reproducibility:} for property-based failures, preserve shrunk/original inputs and configuration where possible.
    \item \textbf{R7 Integration constraints:} wrapper-based interception of failures without changing surrounding tooling.
    \item \textbf{R8 No-solution leakage:} reporting must not expose reference implementations.
\end{itemize}

End with a transition sentence: these requirements define the design goals of Assertify/Checkify.
\end{justify}\end{marker}
