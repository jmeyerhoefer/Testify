\begin{justify}
    This chapter analyses why the feedback produced by compilers and common testing infrastructures is frequently difficult to use in introductory programming settings.
    The focus is not on the correctness of the diagnostics themselves, but on how failure information is \emph{structured}, \emph{prioritised}, and \emph{presented} to learners.
    Based on concrete examples from beginner exercises, the chapter identifies recurring shortcomings in existing output and derives design requirements for a novice-oriented reporting layer.
    These requirements motivate the subsequent design and implementation decisions, in particular the use of a stable \emph{tested expression / expected / actual} summary as
    the primary interface for understanding failures.
\end{justify}
